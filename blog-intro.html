我是一个在韩国做 IVD（体外诊断）研发的中国研究员，合作单位包括 Gachon 大学和 KRIBB（韩国生命工学研究院）。日常工作涉及传染性病原体检测、引物设计、测序数据分析等。为了系统性地提升自己的生物信息学能力，我用 Claude Code 搭建了一个完全免费的 21 天自学网站 —— 这篇文章就是关于它是怎么来的、里面有什么、以及你怎么用它。

为什么要做这个网站？

做 IVD 研发的人，或迟或早都会碰到这些场景：

  • 需要从 NCBI 拉一个病原体参考基因组，设计引物
  • 拿到测序数据，要做比对、变异检测、构建系统发育树
  • 老板让你跑一个 DESeq2 差异表达分析，你连 R 都没装过
  • 论文里的 Snakemake pipeline 看不懂，复现不了
  • 想用 AlphaFold 预测一个抗原蛋白的结构，不知道从哪开始

市面上的生物信息学教程要么太学术（从算法推导开始），要么太碎片（一篇博客解决一个问题），很少有一个系统的、面向实际工作场景的、21 天能走完的自学路径。

所以我决定自己做一个。

网站概览：21 天能学到什么？

21 天课程

整个课程分三周，每周一个主题：

Week 1 · 基础篇（Day 1–7）

从「什么是生物信息学」开始，涵盖：

  • Day 1 — 环境搭建：安装 R + RStudio + Miniconda + Biopython
  • Day 2 — 分子生物学速成：中心法则、DNA/RNA/蛋白质、密码子
  • Day 3 — 文件格式：FASTA、FASTQ、SAM/BAM、VCF、BED、GFF
  • Day 4 — 命令行技能：bash、grep、awk、samtools
  • Day 5 — 公共数据库：NCBI、Ensembl、UniProt、PDB
  • Day 6 — 序列比对与 BLAST
  • Day 7 — 多序列比对与系统发育树

Week 2 · R 语言生物信息学（Day 8–14）

用 R + Bioconductor 做真实的生物数据分析：

  • Day 8 — R 基础 + Bioconductor 生态系统
  • Day 9 — 数据清洗：dplyr + data.table
  • Day 10 — ggplot2 可视化：火山图、热图、箱线图
  • Day 11 — Biostrings + GenomicRanges：序列与区间操作
  • Day 12 — DESeq2 差异表达分析（RNA-seq 核心技能）
  • Day 13 — Seurat 单细胞 RNA-seq 分析
  • Day 14 — 变异注释与 GWAS

Week 3 · Python + 专业级（Day 15–21）

Python 生态 + 生产级工具链：

  • Day 15 — Python 基础 + Biopython
  • Day 16 — Pandas 处理基因组表格数据
  • Day 17 — Snakemake / Nextflow 可复现 pipeline
  • Day 18 — 机器学习（scikit-learn）应用于组学数据
  • Day 19 — 深度学习：AlphaFold / ESM 蛋白质预测
  • Day 20 — 可复现性：Docker、conda、Git
  • Day 21 — 毕业项目：端到端病原体检测 pipeline

IVD 研发视角：为什么这个课程不一样？

大多数生物信息学教程面向的是「学术研究」或「求职」。这个网站的独特之处在于，它是围绕 IVD（体外诊断）研发场景设计的。

具体来说：

IVD 研发场景对应课程内容
设计 PCR 引物前的序列分析 Day 1（获取参考基因组）、Day 6（BLAST 验证特异性）、Day 11（Biostrings 批量筛选引物窗口）
测序数据质控与比对 Day 3（文件格式）、Day 4（命令行 + samtools）、Case Study Level 1（FASTQ QC）
病原体变异监控（引物结合位点突变） Day 14（变异注释）、Case Study Level 3（引物结合位点监控）
宿主应答基因表达分析 Day 12（DESeq2）、Day 18（ML 分类器：病毒 vs 细菌感染）
新抗原蛋白结构预测（免疫检测试剂设计） Day 19（AlphaFold / ESM / ColabFold）
生产级 pipeline + 法规审计 Day 17（Snakemake）、Day 20（Docker + 可复现性）、Day 21（毕业项目）
疫情暴发时的快速建树分型 Day 7（MSA + 系统发育 + Pangolin / Nextstrain）
💡 关键点：每一天的 Case Study 都直接关联一个 IVD 研发中的真实问题。不是「学完了再想怎么用」，而是「学的时候就知道这个技能在工作中怎么落地」。

网站的技术栈与特色功能

纯静态，零依赖

整个网站是纯 HTML + CSS + JavaScript，没有任何框架、没有 Node.js、没有构建步骤。直接丢到 GitHub Pages 就能跑。这意味着：

  • 加载速度极快（没有 React/Vue 的 bundle）
  • 永远不会过时（没有依赖要更新）
  • 任何人都能 fork 并修改

暗色 / 亮色主题切换

右上角一键切换。做生信的人，大部分时间都在暗色终端里工作，这个网站也默认暗色。

学习进度追踪

首页有进度条，基于 localStorage，数据不上传、不泄露。完成一天就打个勾，21 天看着进度条走满，很有成就感。

交互式测验

每天 5 道选择题，点选后立即显示对错和详细解析。总共 105 道题，覆盖所有核心概念。还有一个 Quiz Arena 页面可以随机刷题。

链接永不过期

视频资源用的是 YouTube 搜索链接（而非特定视频 URL），这样即使某个视频被删了，搜索结果里依然能找到相同主题的最新内容。

用 Claude Code 搭建的过程

这个网站从零开始，全部用 Claude Code 在终端里完成。过程大致如下：

  1. 设计课程大纲 — 我告诉 Claude 我的背景（IVD 研发、传染病病原体）、目标（21 天从零到专业级），Claude 帮我规划了三周的课程结构，每天的主题、学习目标、推荐资源。
  2. 生成课程数据 — 21 天的所有内容（objectives、视频、阅读材料、动手任务、测验题、GitHub 仓库、Reddit 社区、案例研究）都是结构化的 JavaScript 数据，存在一个 days-data.js 文件里。
  3. 构建页面 — 首页、每日学习页、资源页、案例研究页、测验竞技场、博客页，每个页面都是独立的 HTML 文件，共用一套 CSS 样式。
  4. 一键部署 — 用 gh CLI 创建 GitHub 仓库，添加 GitHub Actions workflow，自动部署到 GitHub Pages。

整个过程没有打开过浏览器编辑器，全部在终端对话中完成。Claude Code 的优势在于：

  • 领域知识 — 它知道 Bioconductor、DESeq2、Snakemake 这些工具的正确用法，生成的代码示例和测验题质量很高
  • 全栈能力 — 从课程设计到 HTML/CSS/JS 编写到 GitHub Pages 部署，一个工具搞定
  • 迭代速度 — 想改什么直接说，不需要自己翻文件找位置

谁适合用这个网站？

你是谁适合程度建议
IVD / 分子诊断研发人员 完美匹配 这个网站就是为你设计的，Case Study 直接对应你的工作场景
生物学 / 医学背景，想入门生信 非常适合 Week 1 的生物学速成你可以快速跳过，直接进入 R / Python
计算机背景，想转生信 适合 Day 2 的生物学基础对你特别重要，其他代码部分你会学得很快
在读研究生 / 博士生 适合 把每天的任务当作练习，把最后的毕业项目作为你 GitHub portfolio 的起点
已经很熟练的生信工程师 可参考 可以跳到 Week 3 看 ML / AlphaFold / pipeline 部分，或者直接用 Case Studies 作为教学材料

如何开始使用？

方式一：直接在线访问

打开浏览器，访问：

mrfeixiang.github.io/bioinformatics-21days

点击 "Start Day 1" 就可以开始了。

  2. 修改 assets/days-data.js 来调整课程内容
  3. 推送到你自己的 GitHub Pages

git clone https://github.com/mrfeixiang/bioinformatics-21days.git
cd bioinformatics-21days
# 用任何 HTTP 服务器打开，或者直接双击 index.html
python -m http.server 8000

我的学习建议

  1. 不要跳过 Day 1 的环境搭建 — 很多人卡在第一步就放弃了。花 30 分钟把 R、Python、VS Code 装好，后面 20 天才能顺利进行。
  2. 每天真的动手敲代码 — 看视频和读文档只能让你「觉得自己会了」。只有把命令敲进终端、看到输出、遇到错误、解决错误，才是真的学会了。
  3. 做测验 — 105 道题不是摆设。答错了才知道哪里理解有偏差。
  4. 用 Claude Code 当助教 — 遇到不理解的概念、报错信息、或者想扩展某个任务，直接在终端问 Claude。它了解这些工具的上下文，回答质量比 Google 搜索高很多。
  5. 完成 Day 21 的毕业项目 — 这是你 GitHub portfolio 最有价值的一个项目。一个端到端的病原体检测 pipeline，从 FASTQ 到报告，用 Snakemake 编排，Docker 容器化 —— 这就是面试时能拿出来讲的东西。
