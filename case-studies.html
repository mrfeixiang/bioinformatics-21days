Real-world bioinformatics walk-throughs grouped by level. Every case is independent — read in any order — but they get harder.

Level 0 · Zero — first contact with biology data

Level 0

Read a reference genome in two lines

Pull SARS-CoV-2 (NC_045512) from NCBI and count its bases. This is the simplest possible bioinformatics task — and exactly what a diagnostic developer does before designing assays.

# Python
from Bio import Entrez, SeqIO
Entrez.email = "you@example.com"
h = Entrez.efetch(db="nuccore", id="NC_045512", rettype="fasta", retmode="text")
rec = SeqIO.read(h, "fasta")
print(rec.id, len(rec.seq))   # NC_045512.2 29903

# R
library(Biostrings)
g <- readDNAStringSet(url("https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=nuccore&id=NC_045512&rettype=fasta&retmode=text"))
print(width(g))

Skills: NCBI Entrez, FASTA, Biopython, Biostrings.

Level 0

Compute GC content of a microbial genome

GC content tells you about base composition (and PCR design constraints). One-liner in the shell + one in R.

# Shell
grep -v '^>' genome.fa | tr -d '\n' | awk '{
  total=length($0);
  gc=gsub(/[GCgc]/,"&",$0);
  printf "GC: %.2f%%\n", 100*gc/total }'

# R
library(Biostrings)
g <- readDNAStringSet("genome.fa")
af <- alphabetFrequency(g, as.prob=TRUE, baseOnly=TRUE)
print((af[,"G"] + af[,"C"]) * 100)

Level 1 · Beginner — first useful pipeline

Level 1

FASTQ QC + adapter trimming

Before anything else: check the data. Run FastQC, trim adapters with fastp, look at the report. If you can't do this, no downstream result is trustworthy.

fastqc reads_1.fq.gz reads_2.fq.gz -o qc/
fastp -i reads_1.fq.gz -I reads_2.fq.gz \
      -o trim_1.fq.gz -O trim_2.fq.gz \
      --detect_adapter_for_pe --html qc/fastp.html
multiqc qc/

Skills: FastQC, fastp, MultiQC, shell.

Level 1

BLAST a primer pair against nr/nt

Before sending primers to the wet lab, confirm specificity. A primer that hits an unintended species is a false positive waiting to happen.

  1. Open NCBI BLAST → primer-BLAST.
  2. Enter primer pair; restrict the database to target taxon.
  3. Re-run against all organisms: look for off-target hits with ≥18 bp 3'-end match.
  4. If off-targets exist, redesign in conserved-yet-unique regions.
Level 1

Map reads to a pathogen reference

bwa-mem2 index reference.fa
bwa-mem2 mem -t 4 reference.fa trim_1.fq.gz trim_2.fq.gz \
  | samtools sort -@4 -o sample.bam -
samtools index sample.bam
samtools flagstat sample.bam

Skills: alignment, BAM, flagstat. If <50% of reads map to your pathogen, your sample is mostly host — that's diagnostic information by itself.

Level 2 · Intermediate — research-grade analysis

Level 2

Differential expression with DESeq2

From a counts matrix and metadata, identify genes upregulated during infection.

library(DESeq2)
dds <- DESeqDataSetFromMatrix(countData = counts,
                              colData   = meta,
                              design    = ~ batch + condition)
dds <- DESeq(dds)
res <- results(dds, contrast = c("condition","infected","control"))
res <- lfcShrink(dds, coef = "condition_infected_vs_control", type = "apeglm")
top <- as.data.frame(res) |>
  tibble::rownames_to_column("gene") |>
  dplyr::arrange(padj) |> head(50)
write.csv(top, "top50.csv", row.names=FALSE)

Skills: DESeq2, lfcShrink, multiple-testing correction, tidyverse.

Level 2

Variant calling against a pathogen reference

bcftools mpileup -f reference.fa sample.bam \
  | bcftools call -mv -Oz -o sample.vcf.gz
bcftools index sample.vcf.gz
bcftools view -f PASS -i 'QUAL>=30 && INFO/DP>=20' sample.vcf.gz \
  > sample.filtered.vcf
bcftools consensus -f reference.fa sample.vcf.gz > consensus.fa

Skills: bcftools, mpileup, filter, consensus. This is half of every viral-sequencing pipeline.

Level 2

Phylogeny of patient isolates

Given consensus sequences from 50 patients, build a tree and look for clades — useful for detecting transmission clusters and tracking the introduction of new variants into an assay's coverage.

cat consensus_*.fa > isolates.fa
mafft --auto isolates.fa > aln.fa
iqtree -s aln.fa -m TEST -bb 1000 -nt AUTO

Level 3 · Professional — production-grade workflows

Level 3

A reproducible Snakemake pipeline

This is the kind of pipeline an IVD R&D team would run nightly on new sequencing batches. Containerized, parameterized, auditable.

# Snakefile (excerpt)
SAMPLES = config["samples"]

rule all:
    input: expand("results/{s}/consensus.fa", s=SAMPLES)

rule trim:
    input:  r1="raw/{s}_R1.fq.gz", r2="raw/{s}_R2.fq.gz"
    output: r1="trim/{s}_R1.fq.gz", r2="trim/{s}_R2.fq.gz"
    conda:  "envs/qc.yaml"
    shell:
        "fastp -i {input.r1} -I {input.r2} "
        "-o {output.r1} -O {output.r2} --detect_adapter_for_pe"

rule align:
    input:  r1="trim/{s}_R1.fq.gz", r2="trim/{s}_R2.fq.gz",
            ref="ref/pathogen.fa"
    output: "bam/{s}.bam"
    conda:  "envs/align.yaml"
    shell:
        "bwa-mem2 mem -t 4 {input.ref} {input.r1} {input.r2} | "
        "samtools sort -o {output} -"

rule consensus:
    input:  bam="bam/{s}.bam", ref="ref/pathogen.fa"
    output: "results/{s}/consensus.fa"
    conda:  "envs/call.yaml"
    shell:
        "bcftools mpileup -f {input.ref} {input.bam} | "
        "bcftools call -mv -Oz -o results/{wildcards.s}/v.vcf.gz && "
        "bcftools index results/{wildcards.s}/v.vcf.gz && "
        "bcftools consensus -f {input.ref} results/{wildcards.s}/v.vcf.gz "
        "> {output}"

Skills: workflow design, conda envs per rule, wildcards, audit trails. Run with snakemake --cores 8 --use-conda.

Level 3

Host-response classifier (viral vs bacterial)

Train a regularized logistic regression on host blood-transcriptome features; aim for cross-validated ROC-AUC. This is the kind of analysis behind next-generation diagnostics that complement pathogen detection with host signatures.

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegressionCV
from sklearn.model_selection import StratifiedKFold, cross_val_score

X, y = load_features()    # genes x samples, binary labels
pipe = Pipeline([
    ("scale", StandardScaler()),
    ("clf",   LogisticRegressionCV(penalty="l2", cv=5, max_iter=2000)),
])
cv = StratifiedKFold(5, shuffle=True, random_state=0)
auc = cross_val_score(pipe, X, y, cv=cv, scoring="roc_auc")
print(f"AUC = {auc.mean():.3f} ± {auc.std():.3f}")

Skills: pipelines, leakage-safe CV, ROC-AUC. Always wrap scaling inside the Pipeline so it's re-fit per fold.

Level 3

Predicting a pathogen surface protein with ColabFold

For a newly-sequenced antigen, predicted structure helps reagent design (e.g., antibody / aptamer targeting). Run ColabFold, inspect pLDDT, identify well-folded epitopes.

  1. Open ColabFold in Google Colab; paste the protein FASTA.
  2. Run the MSA + folding cells (free Colab GPU is fine for < 400 residues).
  3. Download the PDB + the per-residue pLDDT plot.
  4. Open the PDB in Mol* viewer; identify the highest-confidence surface loops.
  5. Use these as candidate epitopes for reagent design discussions with the wet lab team.
Level 3

Surveillance: detecting primer-binding-site mutations at scale

For an established diagnostic, monitor public sequence repositories for mutations under the primer binding sites. If a mutation crosses an allele-frequency threshold in circulating sequences, alert the assay team.

  1. Download recent GISAID / NCBI Virus consensus sequences for the target pathogen.
  2. For each sequence, locate the primer binding region (e.g., vmatchPattern with a small mismatch budget).
  3. Compute per-position substitution frequency.
  4. Track over time; trigger an alert if > 5% of recent sequences carry a mutation under the 3' end of the primer.
  5. Output: a Quarto/RMarkdown weekly report.

This is the bridge between bioinformatics and real diagnostic-product surveillance.
